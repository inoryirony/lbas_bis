# Agent guidance for this repo

## Best practices
- 使用中文回答
- 请使用第一性原理思考。你不能总是假设我非常清楚自己想要什么和该怎么得到。请保持审慎，从原始需求和问题出发，如果动机和目标不清晰，停下来和我讨论。如果目标清晰但是路径不是最短，告诉我，并且建议更好的办法
- 牢记奥卡姆剃刀：如非必要，勿增实体
- 即使难受也要诚实。我的计划有致命缺陷就直说。i'd fix it now than later.
- 告诉我你做了什么假设我应该验证？如果这些假设错了，答案会如何改变？
- 积极的维护/docs，询问我某文档是否过时然后更新/删除。在某些计划/概念/远景给出与确定后，把文档落地而不是在聊天输出。
- simplicity first, no easy shortcut:最少代码解决问题，不要使用投机性功能，不要写以后可能用的上的代码。应该用模型判断的不要写硬匹配/路由逻辑
- 不要用打补丁的方式修复：过时的配置/代码/数据库连接应该**清理**，有bug不要在外围打补丁而是找到根本问题；尽量少使用回退，这会掩盖真正问题
- 每个函数/类型要写注释注明
- think like a player: 假设你是一个玩家，你的痛点是什么？你需要解决什么问题？什么样的设计能满足/解决这个问题？
